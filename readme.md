### 目录结构
```
├── package.json
├── public
│   ├── controller
│   |   |── article
│   |   └──
│   ├── middleware (中间件)
│   |   |── error_handler.js (后端统一错误处理)
│   |   └── jsredirect.js (开发环境的静态文件重定向)
│   ├── model (sequelize.js model)
│   |   |── ...
|   |   |── ...
│   ├── public (静态文件,由前端打包生成)
│   |   └── ...
│   ├── service (数据库操作,原则上以对应的 table 命名)
│   |   └── db.js
│   ├── tools (工具箱)
│   |   |── ...
|   |   |── ...
│   ├── view
│   |   └── layout.html (模版)
├── config
|   ├── plugin.js
|   ├── config.default.js
│   ├── config.prod.js (生产配置)
|   ├── config.local.js (开发配置)
|   └── ...
├── docs (文档目录)
|   ├── ...
|   └── ...
└── src (前端目录)//未修改

    ├── components (Pure Components 展示组件 不处理业务逻辑)
    |   └── ...
    ├── models (model负责处理状态数据变化,写法遵循dva官方文档)
    |   |── app.js
    |   └── ...
    ├── routes (Container Components 容器组件, 原则上以路由划分文件夹,特殊情况在下面说明- -)
    |   |── backstage
    |   |      |── index.js (使用connect方法与model绑定)
    |   |      |── ...
    |   |      └── style.less
    |   |── collectcenter
    |   |     └── ...
    |   |── contentmine
    |   |     └── ...
    |   |── customercenter (对应订阅规则编辑界面)
    |   |     └── ...
    |   |── datacenter (对应采集中心的原始内容库界面)
    |   |     └── ...
    |   |── error
    |   |     └── ...
    |   |── frontpage
    |   |     └── ...
    |   |── jobcenter
    |   |     └── ...
    |   |── modeljob (对应模版种子录入界面)
    |   |     └── ...
    |   |── newjob (对应标准种子录入界面)
    |   |     └── ...
    |   |── UIElement
    |   |     └── ...
    |   └── app.js (index route)
    ├── services (请求,原则上以路由分治)
    |   └── ...
    ├── utils (配置和公共函数)
    |   └── ...
    ├── index.html
    ├── index.js (入口文件)
    └── router.js (路由配置,这里注册路由并绑定model,其他未绑定的model需要在index.js全局引入)
```
